如何利用hexo d将其部署到阿里云pages上

最近阿里云开放了pages服务，所以本网站搭个车将网站迁移到阿里云pages上面。本文介绍如何通过hexo d上传到GitHub，然后让阿里云pages拉取github项目完成部署。

可以 👍，完全可以做到：

本地 Hexo 继续用 hexo d → 推静态文件到 GitHub 某个仓库 / 分支 → 阿里云 Pages 只拉这一份已经生成好的静态站。

下面我给你一个推荐架构 + 具体配置步骤。

总体思路（先说人话版）

你可以把整个流程理解成三块：

  1. Hexo 源码仓库（带 Markdown / 主题 / 配置）

    • 只用来写文章、本地调试，不直接给阿里云看。

  2. 静态站点仓库或分支（只存 public 生成结果）

    • hexo d 每次跑完就把 public 的内容推到这里。
    • 这个仓库/分支里只有 index.html、CSS、JS 等静态文件。

  3. 阿里云 Pages

    • 直接绑定这个“静态仓库/分支”，
    • 构建命令设成“啥也不干”，只托管现成的文件就行。

方案一：两个 GitHub 仓库（最清晰）

1. 仓库结构

  • 仓库 A：hexo-blog-src
    存 Hexo 源码（你现在的项目）。

  • 仓库 B：hexo-blog-static
    存生成好的静态站点（只 HTML / CSS / JS 等）。

你本地只 clone 仓库 A；仓库 B 完全交给 hexo d 去更新就行。
